Summary

While working in Web2.0 Innovation labs, TCS, I designed a chrome extension for an enterprise social networking application(knome), as my side project. This was later released as an official product. I am the only designer and developer for this project. I handled everything for this product from user experience design, development to deployment.

To give some background Knome is a enterprise social networking application similar to Microsoft Yammer enabling social interaction in work place. Knome had a web and mobile application.

The problem
How can we make Knome experience better and convenient for the power users.
User Research

I conducted multiple focus groups with the power users to understand what they need and how the content should be presented to them. I found pretty interesting things, here are some of them the users needed.

  • Navigate thought different community/timeline views.
  • Able see multiple community posts at a time.
  • Quickly able to comment/like/share any post.
  • Notification are very important.
Prototypes

I sketched many ideas about how to solve this problem. Most of them are completely wild ideas as I was not thinking about the implementation. We had a functional website which is not solving the power users problem. I was looking at sites like Pintrest, Flipboard, TweetDeck, Hootsuite, Feedient and many more.. Each have one common thing like horizontal scroll. It turns out that with horizontal scroll we can present more content in one place at a time. There were many things that didn't work initially.

Thanks to paper prototypes, I eliminated the designs that didn't work in the initial usability testing.

Design

Showing a lot of content in one screen can be a solution for giving quickly accessible content. At the same time there should be readability to make sure that users is not overwhelmed by the amount of content. So I choose a mix of image and text.

The stream based system will enable the users to craft the screen according to their need and interests. The user can re-arrange, delete any streams that they don't need. Enabling Keyboard shortcuts is one other solution for quickly accessing the content.

On boarding helps the users to get started and gives them a glimpse of how they can use the application.
Keyboard shortcuts are one of the main aspects of flame being successful.
Adding a stream to the app is really straightforward.

For re-arranging streams, I designed a separate modal instead of making the streams itself draggable. Because, Lets say, if the user wants to move the first column to the somewhere at the end. The user will host the first column and keep dragging until it reaches the end, this interaction is time consuming and not accurate. The design I had will give the top level over of the streams and user can easily rearrange things. More over the feedback in the background makes the interaction more intuitive.

Learnings

I learned a lot. As I am the only person working on this project. I did many mistakes throughout the process. Being surrounded by some smart people, gave me a chance to get my work critiqued and an opportunity to analyze those mistakes. There was a huge learning curve for the project. So, I learned

  • To judge which feature should go in which release.
  • Project management. Getting approvals. Releasing a product.
  • Always try to solving the problem. Don't get diverted and design some random feature just because its cool.
